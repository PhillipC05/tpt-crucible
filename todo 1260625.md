# TPT Crucible — Master Task Checklist

---

## Phase 1: The Catalyst & The Swarm (Months 1–6)

### TPT Catalyst — Core IR Compiler
- [ ] Set up mono-repo project structure (Rust workspace + Python packages)
- [ ] Integrate MLIR toolchain and define custom TPT-IR MLIR dialect
- [ ] Implement PyTorch model ingestion (.pt / TorchScript)
- [ ] Implement ONNX model ingestion
- [ ] Implement TensorFlow/SavedModel ingestion
- [ ] Integrate Apache TVM for initial graph optimization pass
- [ ] Implement operator fusion (fuse sequential ops like MatMul + activation into single blocks)
- [ ] Define and implement TPT-IR serialization format (JSON + binary)
- [ ] Write unit tests for ingestion and IR correctness
- [ ] CLI: `tpt-catalyst ingest <model>` → outputs `.tptir` file

### TPT Alloy — Swarm / Microcontroller Module
- [ ] Integrate METIS or KaFFPa C++ graph partitioning library
- [ ] Build Python bindings for the partitioning library
- [ ] Implement TPT-IR → neural network graph conversion for partitioning
- [ ] Implement topology-aware partitioning (accept 2D grid / star / custom wiring layout as input)
- [ ] Build Rust-based per-node firmware code generator (C++/Rust output)
- [ ] Integrate PlatformIO build system for targeting ESP32 / RP2040 / RISC-V
- [ ] Optionally integrate Zephyr RTOS support for custom RISC-V targets
- [ ] Generate master flashing script (flash all N nodes from one command)
- [ ] Test partition + firmware gen against TinyLlama on 16x ESP32 swarm
- [ ] CLI: `tpt-alloy partition <model.tptir> --topology grid2d --nodes 16`

### Phase 1 Milestone
- [ ] **DEMO:** Load TinyLlama → TPT Catalyst → TPT Alloy → flash 16x ESP32 → run basic chat interface

## Phase 3: The Physics Engine (Year 2)

### TPT Element — Analog Compute Module
- [ ] Set up Xyce SPICE simulator (preferred) with PySpice Python wrapper
- [ ] Add ngspice fallback support
- [ ] Implement floating-point weight → physical component mapping (resistors, memristors, op-amp gains)
- [ ] Build SPICE netlist generator from TPT-IR weights
- [ ] Implement thermal noise injection in simulations
- [ ] Implement voltage drift simulation
- [ ] Implement component tolerance error simulation (1%, 5%, 10% tolerance profiles)
- [ ] Build "Reality Check" dataset: run thousands of SPICE simulations, capture failure modes
- [ ] Train lightweight PyTorch ML model on SPICE dataset to predict thermal/noise drift instantly
- [ ] Integrate Reality Check model as fast inference pass (replacing slow brute-force SPICE for routine checks)
- [ ] Generate human-readable mitigation suggestions (e.g., "Add heatsink to Node 4", "Use 1% resistors here")
- [ ] Generate SPICE netlist output files
- [ ] Generate PCB layout recommendations
- [ ] Integrate KiCad file export for PCB manufacturing
- [ ] Compute and output analog "Confidence Score" for each design
- [ ] Test on a 3-layer analog neural network design

### Phase 3 Milestone
- [ ] **DEMO:** Design 3-layer analog NN → TPT Element simulates thermal drift → outputs KiCad PCB file ready for manufacturing
